# Bradford Portal - 4-Minute Video Presentation Script

## Video Script (4 minutes total)

---

**[Opening - 0:00-0:30]**

*Visual: Fade in with Bradford Portal logo, then show the login page*

"Hello everyone! I'm Haris, and today I'm excited to present our team's final project - the Bradford Portal. This is a comprehensive web application developed by our team to help Bradford Council manage and visualize location-based data. Let me walk you through what we've built."

---

**[Project Overview - 0:30-1:00]**

*Visual: Show main dashboard with map, navigation menu*

"The Bradford Portal is a secure web application that allows users to upload, manage, and visualize geographical data for Bradford. Our system includes:

- User authentication with secure login/registration
- Interactive Google Maps integration
- Data upload and management features
- Admin dashboard for system management
- Advanced security features

Built with PHP, MySQL, and modern web technologies."

---

**[Core Features - 1:00-2:00]**

*Visual: Demonstrate each feature with screen recordings*

**Authentication System:**
"Users can register with email and password, login securely, and reset passwords. We implemented bcrypt hashing and session management."

*Show: Registration form, login page*

**Data Management:**
"Users can upload CSV or JSON files containing location data, or manually enter coordinates. The system parses and stores this data securely."

*Show: Upload form, manual entry, data appearing on map*

**Map Visualization:**
"Using Google Maps API, we display data as interactive markers. Users can toggle between standard markers and heatmaps for data density visualization."

*Show: Map with markers, heatmap toggle*

**Admin Features:**
"Administrators can manage users, view activity logs, and import data from external APIs like car parks and schools."

*Show: Admin dashboard, activity log, API import*

---

**[Advanced Features - 2:00-3:00]**

*Visual: Show each advanced feature*

**Data Sharing:**
"Users can share their data with other users, with view or edit permissions."

*Show: Share data page, shared data on map with yellow markers*

**Audit Trails:**
"Complete version history and audit logging for all data changes."

*Show: Data history page, version downloads*

**Accessibility:**
"High contrast mode, font size adjustment, and ARIA labels for screen readers."

*Show: Accessibility controls in action*

**Security Features:**
"Rate limiting on login attempts, CSRF protection, HTTPS enforcement, and SQL injection prevention."

*Show: Security test page, rate limiting in action*

---

**[Technical Implementation - 3:00-3:30]**

*Visual: Show code snippets, database schema*

"Technically, we used:
- PHP 7.4+ with PDO for database access
- MySQL with encrypted sensitive data
- Google Maps API with visualization library
- PHPMailer for email notifications
- Session management with secure cookies

Our database includes tables for users, uploads, activity logs, data sharing, and version control."

---

**[Production Deployment - 3:30-4:00]**

*Visual: Show production setup guide*

"For production deployment, we've created comprehensive documentation covering:
- Server setup with Apache/Nginx
- SSL certificate configuration
- Database security hardening
- Automated backups and monitoring
- Security best practices

The application is production-ready with enterprise-grade security."

---

**[Closing - 3:50-4:00]**

*Visual: Show final dashboard, team credits*

"This project demonstrates our team's ability to build secure, scalable web applications with modern technologies. Thank you for watching!"

*Fade out with "Developed by Haris and Team"*

---

## Timing Breakdown:
- Introduction: 30 seconds
- Overview: 30 seconds
- Core Features: 1 minute
- Advanced Features: 1 minute
- Technical Details: 30 seconds
- Production Setup: 30 seconds
- Closing: 10 seconds

## Key Points to Emphasize:
1. Team collaboration and planning
2. Security-first approach
3. User experience and accessibility
4. Production readiness
5. Modern technology stack

## Visual Suggestions:
- Use screen recordings, not static images
- Show actual functionality working
- Include code snippets briefly
- Demonstrate user workflows
- Show before/after comparisons
